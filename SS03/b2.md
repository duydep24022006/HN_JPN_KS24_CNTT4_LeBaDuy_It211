Bài tập 2: HTTP Request / Response và các thành phần
1. Các thành phần trong HTTP Request
Cấu trúc HTTP Request gồm:
POST /api/sanpham HTTP/1.1
Host: example.com
Content-Type: application/json
Authorization: Bearer abc123

{
   "ten": "Laptop",
   "gia": 1500
}
Thành phần của Request
Thành phần	Giá trị
Method	POST
URL	/api/sanpham
HTTP Version	HTTP/1.1
Headers	Host, Content-Type, Authorization
Body	Dữ liệu JSON sản phẩm
Giải thích các Header
Host
Host: example.com

Ý nghĩa:

xác định domain của server
cho biết request gửi tới website nào
Content-Type
Content-Type: application/json

Ý nghĩa:

cho server biết kiểu dữ liệu được gửi
ở đây dữ liệu là JSON
Authorization
Authorization: Bearer abc123

Ý nghĩa:

dùng để xác thực người dùng
Bearer Token thường dùng trong REST API và JWT Authentication
Body của Request
{
   "ten": "Laptop",
   "gia": 1500
}

Ý nghĩa:

chứa dữ liệu client gửi lên server
ở đây là thông tin sản phẩm mới
2. Các thành phần trong HTTP Response
Ví dụ Response
HTTP/1.1 201 Created
Content-Type: application/json

{
   "message": "Tạo sản phẩm thành công"
}
Thành phần của Response
Thành phần	Giá trị
HTTP Version	HTTP/1.1
Status Code	201
Status Message	Created
Headers	Content-Type
Body	JSON message
Ý nghĩa mã trạng thái 201 Created
thuộc nhóm: 2xx Success
cho biết request đã được xử lý thành công
tài nguyên mới đã được tạo trên server

Ví dụ:

tạo sản phẩm mới
tạo tài khoản
thêm dữ liệu mới
3. Nếu GET /api/sanpham/999 nhưng sản phẩm không tồn tại

Server sẽ trả về:

404 Not Found
Ý nghĩa
tài nguyên được yêu cầu không tồn tại
server không tìm thấy sản phẩm có id = 999

Thuộc nhóm:

4xx Client Error
4. Khi server gặp lỗi xử lý không xác định

Server thường trả về:

500 Internal Server Error
Ý nghĩa
server xảy ra lỗi trong quá trình xử lý request
lỗi không xác định từ phía server

Thuộc nhóm:

5xx Server Error

Ví dụ:

lỗi code backend
lỗi database
exception chưa xử lý
server crash
